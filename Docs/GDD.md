## 1. 项目概述

### 1.1 一句话 Pitch

> 这是一款 AI 原生 Roguelike 游戏。AI 不再只是内容生成工具，而是化身为游戏导演，通过持续分析玩家的战斗风格，动态设计敌人组合、关卡规则与资源配置，迫使玩家不断突破固有策略，让每一局都成为一场专属于玩家的全新挑战。

### 1.2 核心思想

**AI is the Gameplay.**

不是"AI is a Tool"——去掉 AI，整个游戏核心玩法便不存在。

### 1.3 项目目标

- 参加腾讯游戏开发大赛
- 优先保证完成度，其次展示 AI 原生玩法创新
- 2 个月交付可完整游玩 ~15 分钟的 MVP

### 1.4 团队分工

| 成员 | 职责 |
|---|---|
| 我 | 全部程序开发、系统架构、AI 系统、工具链 |
| 队友 | 美术资源、UI、音效、关卡、宣传素材、文案 |

## 2. 核心设计理念

### 2.1 三大设计支柱

**① AI 理解玩家**

AI 记录玩家整局行为，分析战斗习惯、Build 偏好、操作风格，生成玩家画像。

**② AI 导演游戏**

AI 不是随机生成，而是根据玩家画像设计下一关挑战，让游戏不断针对玩家。

**③ 玩家必须不断改变策略**

不存在固定最优 Build。玩家越依赖一种打法，AI 越会针对这种打法，迫使玩家不断调整策略。

### 2.2 难度≠数值，难度=AI 的人格

参考《生化危机 1》的难度描述，我们设计三种 AI 人格模式：

| 模式 | AI 人格 | 核心行为 | 玩家感受 |
|---|---|---|---|
| **温和模式** | 伙伴 | 主动帮助玩家通关 | "AI 在照顾我" |
| **正常模式** | 裁判 | 循规蹈矩，按部就班 | "公平对决" |
| **整人模式** | 对手 | 处处与玩家作对 | "AI 在针对我" |

**三种模式的规则集不同**（不是数值缩放），后续两个模式在后续版本开放。

**MVP 只做正常模式。**

## 3. 世界观与角色

### 3.1 世界观

**中国风 · 山海经主题**。敌人、Boss、场景、武器、观察者角色均取材自《山海经》。

### 3.2 观察者角色：白泽

白泽是 AI 导演在游戏中的化身。

| 设定 | 内容 |
|---|---|
| 出处 | 《山海经》——通晓万物的神兽，能人言 |
| 外观 | 白色身躯，羊首独角，智者形象 |
| 定位 | AI 导演的化身，关卡预告卡头像 + 战斗旁白 |
| MVP 形式 | 纯文字，后续版本加配音 |

**三种模式下的台词风格**：

| 模式 | 态度 | 举例 |
|---|---|---|
| 温和 | 慈祥导师 | "你的远程用得很好，这层我帮你少放些盾兵。" |
| 正常 | 冷静观察者 | "你的弓用了四层了。这层，试试别的？" |
| 整人 | 毒舌吐槽 | "还射？你是只会这一招吗？" |

### 3.3 可玩角色（MVP：2 个）

| | 力士 | 方士 |
|---|---|---|
| **定位** | 近战重装 | 远程灵修 |
| **武器类型** | 重剑、长刀、拳套 | 符箓、灵幡、卦镜 |
| **专武** | 干戚（刑天之斧） | 八卦镜 |
| **初始技能** | 战吼（减伤 + 嘲讽） | 灵符（远程攻击 + 标记） |
| **战斗风格** | 正面硬刚，弹反格挡 | 走位拉扯，法术消耗 |
| **AI 眼中画像** | "贴脸型" | "风筝型" |

### 3.4 进化系统（后续版本，不在 MVP）

角色可进化为神话形态：
- 力士 → 刑天 / 饕餮 / 应龙
- 方士 → 祝融 / 共工 / 后羿

## 4. 核心玩法循环

### 4.1 整体循环

```
进入关卡 → 战斗 → 记录玩家行为 → 获得强化 → AI 分析玩家 → AI 导演下一关 → 继续挑战
```

### 4.2 AI 分析时机

**每层结束时调用 LLM 分析一次**，不采用实时分析。

原因：
- 降低网络需求
- 控制 API 成本
- 规避延迟风险

## 5. AI 导演系统

### 5.1 三根指挥棒

AI 导演的内容限定为三个方向：

**① 敌人组合（Enemy Composition）**

根据玩家行为调整刷怪概率表。例如：
- 玩家偏远程 → 增加盾兵比例、增加突进敌人、减少普通近战敌人

**② 规则修改（Rule Modifier）** ⭐ 核心创新

AI 不是在改数值，而是在**改规则**。

规则分为三个层级：

| 层级 | 描述 | 正常模式 | 举例 |
|---|---|---|---|
| **轻度** | 数值微调，玩家不一定察觉 | ✅ | 敌人攻速 +10%、冷却延长 1 秒 |
| **中度** | 明显改变玩法，但不封死路线 | ✅ | 弹药掉落减半、回复效果 -30% |
| **重度** | 直接废掉 Build 核心机制 | ❌ | 暴击失效、远程伤害归零 |

可修改的规则示例池：

- 弹药消耗率
- 回复道具效果
- 暴击率/暴击伤害
- 近战/远程伤害倍率
- 技能冷却时间
- 敌人特殊 Buff（护盾/加速/霸体）

**③ 资源分布（Resource Distribution）**

控制关卡内资源供给：
- 回复道具数量
- 宝箱奖励质量
- 精英怪数量

### 5.2 资源分布策略：浮动制

AI 干预力度随关卡深度递增——前几层改动少，越深越多。

### 5.3 规则修改的动态难度

AI 不机械地每层套固定规则，而是基于**玩家行为评分**决定：
- 改什么规则
- 改多大幅度
- 改几条

## 6. 玩家行为评分系统

### 6.1 五个评分维度

| 维度 | 测量方式 | 含义 |
|---|---|---|
| **Build 集中度** | 同一技能/武器的使用频率占比 | 越高 = 越依赖单一打法 |
| **战斗效率** | 清怪速度、受击次数 | 太高 = AI 需加大针对 |
| **资源盈余** | 剩余血瓶/弹药/护盾 | 太多 = AI 可减少补给 |
| **策略切换意愿** | 战斗间是否更换武器/技能 | 低 = 玩家在"一招鲜" |
| **生存压力** | 残血次数、濒死次数 | 太多 = AI 应收一收 |

### 6.2 评分与 AI 响应

每层结束综合评分，输出 **0-100 分**：

| 分数区间 | 玩家状态 | AI 行为 |
|---|---|---|
| **0-30** | 挣扎中 | 减轻规则力度，提高资源掉落，只出基础近战杂兵 |
| **31-60** | 适应中 | 维持当前力度，微调 |
| **61-100** | 太顺了 | 加大规则力度，降低资源供给 |

### 6.3 定向针对映射

高分不是随机加难度，而是**定向针对**——哪个维度分高就针对哪个方向：

| 分数来源 | AI 选择 |
|---|---|
| Build 集中度太高 | 出克制该 Build 的敌人 / 削弱该 Build 的规则 |
| 战斗效率太高 | 增加精英怪、调整波次、提高敌人攻速 |
| 资源太多 | 减少掉落、降低宝箱质量 |
| 策略切换意愿低 | 给中度规则，逼玩家换打法 |

### 6.4 数据采集项（14 项埋点）

**维度 1：Build 集中度**
- 每种武器的攻击次数（每次攻击 +1）
- 每个技能的释放次数（每次释放 +1）
- 每种武器/技能的总伤害量（每次造成伤害时累加）
- 每种武器/技能的击杀数（击杀时记录）

**维度 2：战斗效率**
- 房间清怪耗时（房间结束时）
- 受击次数、总受伤量（每次受击时）
- 闪避成功率（战斗中计算）

**维度 3：资源盈余**
- 回复道具存量（层结束时快照）
- 弹药/能量余量（层结束时快照）

**维度 4：策略切换意愿**
- 战斗中武器切换频率（每次切换 +1）
- 本层使用的不同技能数（层结束时统计）
- Build 与上一层的差异度（层间对比）

**维度 5：生存压力**
- HP < 20% 次数（每次触发 +1）
- 死亡/濒死次数（死亡时记录）

## 7. Build 强化系统

### 7.1 三维强化体系

| 维度 | 槽位 | 数量 | 机制 |
|---|---|---|---|
| **武器** | 主武器 + 副武器 | 6 把 | 共享池 4 + 力士专武 1 + 方士专武 1 |
| **主动技能** | 2-3 个槽 | 6 个 | 替换式，新技能覆盖旧技能 |
| **被动强化** | 不限（叠加） | ~10 个 | 可叠加，每层限换 1-2 个 |

### 7.2 强化来源

| 来源 | 说明 |
|---|---|
| 每层奖励 | 清完一层后从 3 个强化中选 1 个（经典肉鸽） |
| Boss 掉落 | 击败 Boss 获得稀有强化 |

（商店系统和元素系统不在 MVP，后续版本开放。AI 导演不会直接赠送强化。）

### 7.3 Build 切换机制

每层允许替换 1-2 个被动强化，支持玩家在 AI 针对下主动切换打法。

### 7.4 强化内容方向

**武器（6 把）**：
- 共享：青铜剑、猎弓、灵幡、卦镜
- 力士专武：干戚（高伤慢速）
- 方士专武：符箓（远程标记增伤）

**主动技能（6 个）**：
- 战吼、灵符、翻滚、震地、疾风步、灵盾

**被动强化（~10 个）**：
- 攻击力 ↑、暴击率 ↑、最大生命 ↑、击杀回血
- 冷却缩减、近战伤害 ↑、远程伤害 ↑
- 闪避后增伤、低血量增伤、每层开始回血

## 8. 关卡设计

### 8.1 关卡结构

**Slay the Spire 式自由探索**。每层多个房间自由进出，清完所有战斗房后触发通往下一层的出口。

### 8.2 5 层结构

```
第 1 层：入门层（建立初步 Build）
第 2 层：AI 开始观察，轻度调整
第 3 层：精英怪登场
第 4 层：AI 规则加深，压力递增
第 5 层：Boss 层（烛龙）
```

| 参数 | 数值 |
|---|---|
| 总层数 | 5 层 |
| 每层时长 | ~3 分钟 |
| 总时长 | ~15 分钟 |
| Boss 位置 | 第 5 层 |

### 8.3 AI 输出注入管线

```
LLM 输出                          关卡生成器
──────────                       ──────────
敌人配比      ─────────────→     刷怪时按配比概率选怪
规则修改      ─────────────→     全局参数覆盖（伤害/掉落/冷却）
资源分布      ─────────────→     掉落表调整
```

## 9. 敌人设计

### 9.1 按 AI 克制关系分 6 类

| 分类 | 克制目标 | 行为特征 | 山海经取材方向 |
|---|---|---|---|
| **近战杂兵** | 无（基础） | 走来平 A | 魑魅、山魈 |
| **盾兵** | 远程/方士 | 正面无敌，缓慢推进 | 玄龟、石敢当 |
| **突进怪** | 远程/方士 | 快速冲脸，打断施法 | 穷奇、蛊雕 |
| **远程怪** | 近战/力士 | 远程射击，近身会跑 | 朱厌、鸣蛇 |
| **控场怪** | 力士（限位） | 减速/定身/拉拽 | 藤妖、冰夷 |
| **精英怪** | 综合考验 | 多技能，有词条 | 狰、毕方、九尾狐 |

### 9.2 数量分配

MVP 总计 12-15 种，具体异兽选型由队友填充。

### 9.3 AI 导演使用逻辑

- 行为分显示"远程占比高" → 盾兵 +30%、突进怪 +20%
- 行为分显示"贴脸站撸" → 远程怪 +30%、控场怪 +20%
- 行为分 0-30（挣扎） → 只出近战杂兵，给喘息空间

| 设定 | 内容 |
|---|---|

## 12. 技术方案

### 12.1 引擎

首选 **UE5**，若最终 Scope 过大再考虑 Unity。

### 12.2 AI 管线

```
采集行为数据 → 本地预处理(算行为分) → LLM 决策(选规则+幅度+文案) → 应用到下一层
```

### 12.3 LLM 调用

| 参数 | 内容 |
|---|---|
| API | 腾讯混元 / GameNPC（官方提供） |
| 调用频率 | 每层结束调用一次 |
| 调用方式 | 轻量调用，不训练模型 |
| 兜底方案 | 本地预设规则表，API 失败时按行为分查表自动出规则 |

### 12.4 LLM 接口定义

**输入**：
```json
{
  "player_profile": {
    "build_concentration": 87,
    "combat_efficiency": 92,
    "resource_surplus": 65,
    "strategy_switch": 20,
    "survival_pressure": 10
  },
  "current_floor": 4,
  "total_floors": 5,
  "active_rules": ["弹药掉落 -20%"],
  "player_build": ["猎弓", "远程暴击流"],
  "character": "方士"
}
```

**输出**：
```json
{
  "enemy_composition": [
    { "type": "盾兵", "ratio": "+30%" },
    { "type": "突进怪", "ratio": "+20%" }
  ],
  "rule_modifiers": [
    { "rule": "弹药掉落", "adjustment": "-30%", "level": "medium" }
  ],
  "resource_changes": [
    { "resource": "治疗道具", "adjustment": "-15%" }
  ],
  "narrator_line": "你的弓耍得不错，但这层可不会让你这么舒服了。"
}
```

### 12.5 架构要求

Rule Modifier 系统设计为**可配置规则池**，后续只需往池子加规则 + 改 AI 人格参数即可切换模式。规则不写死在代码里。
