## 1. MSVC 14.44 不兼容 UE 5.5

### 现象

```
c1xx: fatal error C1083: Cannot open compiler intermediate file: 
"SharedPCH.UnrealEd.Project.ValApi.Cpp20.h.pch": No such file or directory
```

### 原因

UE 5.5 的预编译头（PCH）机制依赖 MSVC **14.38.33130**。VS 2022 默认安装的最新工具链是 14.44，UnrealBuildTool 会自动选择最新可用版本，导致 PCH 编译失败。

### 解决方案

1. 打开 **Visual Studio Installer** → 修改 → 单个组件
2. 搜索并安装：`MSVC v143 - VS 2022 C++ x64/x86 build tools (v14.38-17.8)`
3. 创建 `%APPDATA%\Unreal Engine\UnrealBuildTool\BuildConfiguration.xml`：

```xml
<?xml version="1.0" encoding="utf-8" ?>
<Configuration xmlns="https://www.unrealengine.com/BuildConfiguration">
    <WindowsPlatform>
        <CompilerVersion>14.38.33130</CompilerVersion>
    </WindowsPlatform>
</Configuration>
```

### 验证

```powershell
# 查看已安装的 MSVC 版本
ls "C:\Program Files\Microsoft Visual Studio\2022\Community\VC\Tools\MSVC"
```

确认 `14.38.33130` 目录存在。

## 2. 中文路径导致 MSVC 无法写入中间文件

### 现象

同上 C1083 错误，即使 MSVC 版本正确也无法解决。

### 原因

MSVC 编译器对含中文字符的路径处理有问题，无法在中文路径下创建 `.pch` 预编译头文件和 `.obj` 中间文件。

### 解决方案

将整个 UE5 工程放在**纯 ASCII 路径**下。

❌ 不可用：
```
C:\Users\徐傲轩\Desktop\游戏开发\ShanHaiMirror\
```

✅ 可用：
```
C:\Dev\ShanHaiMirror\
```

### 规则

UE5 工程的**完整路径**中不得包含任何非 ASCII 字符（中文、日文、特殊符号等）。

## 3. UE5 License 未激活警告（可忽略）

### 现象

```
Build system warning: License not activated
```

### 原因

UnrealBuildTool 的命令行编译模式不检查 Epic Games 的登录状态，即使编辑器能正常打开也会显示此警告。

### 结论

**不影响编译和运行**，可以忽略。
